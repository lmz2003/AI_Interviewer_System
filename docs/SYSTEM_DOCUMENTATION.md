## 1. 系统概述

### 1.1 系统定位

本系统是一款面向求职者的**智能求职辅助平台**，集成了笔记管理、简历智能分析、知识库构建、AI智能助手等核心功能。系统采用前后端分离架构，基于现代化技术栈构建，旨在为求职者提供一站式的求职准备工具。

系统通过整合大语言模型（LLM）、向量数据库（Milvus）和RAG（检索增强生成）技术，实现了智能化的简历分析、知识问答和面试模拟功能，帮助用户提升求职竞争力。

### 1.2 目标用户

| 用户群体 | 使用场景 | 核心需求 |
|---------|---------|---------|
| **应届毕业生** | 校园招聘准备 | 简历优化、面试技巧学习、知识点整理 |
| **社招求职者** | 职业转型/跳槽 | 简历针对性优化、岗位匹配分析、技能梳理 |
| **在校学生** | 实习申请 | 简历制作入门、知识积累、面试模拟 |
| **HR/招聘人员** | 简历筛选参考 | 简历质量评估、候选人能力分析 |

## 2. 功能模块详细说明

### 2.1 用户认证模块

#### 2.1.1 功能概述

用户认证模块基于GitHub OAuth实现单点登录，采用JWT（JSON Web Token）进行身份验证和授权管理。

#### 2.1.2 功能点说明

| 功能 | 描述 | API端点 |
|------|------|---------|
| GitHub登录 | 通过GitHub OAuth授权登录 | `POST /api/auth/github` |
| Token验证 | 验证JWT Token有效性 | 中间件自动处理 |
| 用户信息获取 | 获取当前登录用户信息 | `GET /api/users/me` |

#### 2.1.3 操作流程

```
用户点击"使用GitHub登录"
        │
        ▼
┌───────────────────┐
│  跳转GitHub授权页  │
│  用户授权应用访问  │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  GitHub回调       │
│  返回授权码(code) │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  前端发送code     │
│  到后端验证       │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  后端换取access_  │
│  token并获取用户  │
│  信息             │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│  生成JWT Token    │
│  返回给前端存储   │
└───────────────────┘
```

### 2.2 笔记管理模块

#### 2.2.1 功能概述

笔记管理模块提供完整的笔记创建、编辑、管理功能，支持富文本编辑、版本控制、评论协作，并可一键同步至知识库。

#### 2.2.2 功能点说明

| 功能 | 描述 | API端点 | 方法 |
|------|------|---------|------|
| 创建笔记 | 创建新的笔记文档 | `/api/notes` | POST |
| 获取笔记列表 | 分页获取用户笔记 | `/api/notes` | GET |
| 获取笔记详情 | 获取单篇笔记内容 | `/api/notes/:id` | GET |
| 更新笔记 | 修改笔记内容 | `/api/notes/:id` | PUT |
| 删除笔记 | 删除指定笔记 | `/api/notes/:id` | DELETE |
| 批量删除 | 批量删除多篇笔记 | `/api/notes` | DELETE |
| 获取版本历史 | 查看笔记历史版本 | `/api/notes/:id/versions` | GET |
| 恢复历史版本 | 恢复到指定版本 | `/api/notes/:id/restore/:versionId` | POST |
| 获取评论 | 获取笔记评论列表 | `/api/notes/:id/comments` | GET |
| 添加评论 | 添加评论或回复 | `/api/notes/:id/comments` | POST |
| 删除评论 | 删除指定评论 | `/api/notes/:id/comments/:commentId` | DELETE |
| 上传至知识库 | 首次上传笔记到知识库 | `/api/notes/:id/upload-to-knowledge` | POST |
| 同步至知识库 | 更新知识库中的笔记 | `/api/notes/:id/sync-to-knowledge` | POST |

#### 2.2.3 笔记数据模型

```typescript
interface Note {
```

### 2.3 简历分析模块

#### 2.3.1 功能概述

简历分析模块是本系统的核心功能之一，利用大语言模型对用户上传的简历进行多维度智能分析，提供专业的优化建议和岗位匹配度评估。

#### 2.3.2 功能点说明

| 功能 | 描述 | API端点 | 方法 |
|------|------|---------|------|
| 上传简历 | 上传PDF/Word/文本简历 | `/api/resume-analysis/upload` | POST |
| 获取简历列表 | 获取用户所有简历 | `/api/resume-analysis` | GET |
| 获取简历详情 | 获取单份简历详情 | `/api/resume-analysis/:id` | GET |
| 获取简历PDF | 获取简历原始文件 | `/api/resume-analysis/:id/pdf` | GET |
| 分析简历 | 触发AI分析 | `/api/resume-analysis/:id/analyze` | POST |
| 获取分析结果 | 获取分析报告 | `/api/resume-analysis/:id/analysis` | GET |
| 删除简历 | 删除简历及分析 | `/api/resume-analysis/:id` | DELETE |

#### 2.3.3 支持的简历格式

| 格式 | 扩展名 | 解析方式 |
|------|--------|---------|
| PDF | .pdf | pdf-parse库 |
| Word | .docx, .doc | mammoth库 |
| 文本 | .txt | 直接读取 |
| 最大文件大小 | - | 10MB |

#### 2.3.5 分析维度与评分体系

系统采用多维度评分体系，针对校招和社招采用差异化评估标准：

```
┌─────────────────────────────────────────────────────────────────┐
│                      简历分析评分体系                            │
├─────────────────────────────────────────────────────────────────┤

│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  📋 完整性评分 (Completeness Score)                             │
│  ├── 校招权重：个人信息(10%) + 教育背景(25%) + 实习经历(20%)    │
│  │             + 技能(15%) + 项目经验(15%) + 校园经历(15%)      │
│  └── 社招权重：个人信息(10%) + 工作经验(35%) + 技能(25%)        │
│               + 项目经验(20%) + 教育背景(10%)                   │
├─────────────────────────────────────────────────────────────────┤
│  🔑 关键词评分 (Keyword Score)                                  │
│  ├── 技术关键词匹配度                                           │
│  ├── 行业术语覆盖度                                             │
│  └── 岗位相关关键词密度                                         │
├─────────────────────────────────────────────────────────────────┤
│  💼 经验评分 (Experience Score)                                 │
│  ├── 校招：实习质量、项目深度、校园活动                         │
│  └── 社招：工作年限、职业发展轨迹、管理经验                     │
├─────────────────────────────────────────────────────────────────┤
│  🛠️ 技能评分 (Skills Score)                                    │
│  ├── 技能广度与深度                                            │
│  ├── 技能与岗位匹配度                                          │
│  └── 技能认证与证明                                            │
├─────────────────────────────────────────────────────────────────┤
│  📊 岗位匹配分析 (Job Match Analysis) - 可选                    │
│  ├── 岗位匹配度评分                                            │
│  ├── 匹配技能列表                                              │

│  └── 针对性改进建议                                            │
├─────────────────────────────────────────────────────────────────┤
│  🎯 能力素质评估 (Competency Analysis)                          │
│  ├── 核心能力素质                                              │
│  ├── 技术技能等级                                              │

│  └── 职业发展潜力                                              │
```

### 2.4 知识库模块

#### 2.4.1 功能概述

知识库模块基于RAG（检索增强生成）技术构建，支持用户上传和管理个人知识文档，通过向量化和语义检索实现智能问答。

#### 2.4.2 功能点说明

```
│     ├── Payload: { userId, githubId, iat, exp }                │
│     ├── 签名算法: HS256                                         │
│     └── 有效期: 可配置（默认24小时）                            │
│                                                                 │
│  2. 前端存储Token                                               │
│     └── localStorage.setItem('token', jwtToken)                │
│                                                                 │
│  3. 请求携带Token                                               │
│     └── Authorization: Bearer <token>                          │
│                                                                 │
│  4. 服务端验证                                                  │
│     ├── JwtStrategy 解析验证Token                               │
│     ├── 提取用户信息注入Request                                 │
│     └── 失败返回401 Unauthorized                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 6. 部署与配置说明

### 6.1 环境要求

| 组件 | 版本要求 |
|------|---------|
| Node.js | ≥22.0.0 |
| PostgreSQL | ≥14 |
| Milvus | latest |
| Docker | 可选，用于容器化部署 |

### 6.2 环境变量配置

**后端环境变量 (.env)：**

```bash
# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=your_password
DB_NAME=notes

# Milvus配置
MILVUS_HOST=localhost
MILVUS_PORT=19530

# OpenAI配置
OPENAI_API_KEY=sk-your-api-key

# JWT配置
JWT_SECRET=your-jwt-secret
JWT_EXPIRES_IN=24h

# GitHub OAuth配置
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret

# 请求限制
MAX_REQUESTS_PER_15MIN=5
```

**前端环境变量 (.env)：**

```bash
VITE_API_BASE_URL=http://localhost:3001/api
```

### 6.3 Docker部署

使用docker-compose一键部署所有服务：

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 停止所有服务
docker-compose down
```

**Docker Compose服务列表：**

| 服务名 | 端口 | 说明 |
|-------|------|------|
| postgres | 5432 | PostgreSQL数据库 |
| milvus | 19530, 9091 | Milvus向量数据库 |
| etcd | 2379 | Milvus依赖 |
| minio | 9000, 9001 | Milvus对象存储 |
